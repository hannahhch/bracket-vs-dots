How are they similar?

They are the two ways we can access Javascript Object properties


	let myInformation = {
		firstName: "Hannah"
		lastName: "Hudson",
		birthday: "September 23rd",
	};

	console.log(hannah.lastName);

	prints Hudson

	console.log(hannah["lastName"]);

	prints Hudson

Which syntax is your favorite?

Advantages to Dots

Advantages to Dots

  • Popular option is that its easier to read
  • Fewer characters, might save some time

Disadvantages to Dots

We can't do stuff like this:

	let importantDates = {
		711: "My cat's birthday",
		928: "My wedding anniversary",
	};

	console.log(importantDates.711);
Uncaught SyntaxError: missing ) after argument list

Disadvantages to Dots

We can't access properties when a key contains a space

	let importantDates = {
	  "My cat's birthday": 711,
	  "My wedding anniversary": 928,
	};

	console.log(importantDates."My cat's birthday");
Uncaught SyntaxError: Unexpected string

Disadvantages to Dots

We can't access arrays

	const favBakedGoods = ['tarts', 'pies', 'cakes'];

	console.log(favBakedGoods.0);
Uncaught SyntaxError: missing ) after argument list

Disadvantages to Dots

We can't loop or map over Object keys

	let myFamily = {
		husband: "Erik",
		cat: "Clementine",
	}

	for (let key in myFamily) {
	  let value = myFamily.key;
	  console.log("My", key, "'s name is ", value);
	}

	prints
	My husband 's name is undefined
	My cat 's name is undefined

Why is dot notation so limited?

According to MDN, the property (value) must be a valid JavaScript identifier (must start with a letter, underscore (_), or dollar sign ($).

Advantages to Brackets

Advantages to Brackets

  • Can access data without knowing what it is beforehand (think of APIs!)
    • 
		let mysteryData = response.json();

		for (let key in mysteryData) {
		 console.log(mysteryData[key])
		}

		print 
			"value 1"
			"value two"
			"other mystery value"
			"more"

    Advantages to Brackets

  • Can access number keys, keys with spaces, hyphens and more!
    • 
	let importantDates = {
		"My cat's birthday": 711,
		"My wedding anniversary": 928,
	};

	console.log(importantDates["My cat's birthday"]);

	prints 711

    Advantages to Brackets

    We can use variables to access Object properties
    
		const today = 'wednesday';
		
		const weather = {
		 monday: 'sunny',
		 tuesday: 'rainy',
		 wednesday: 'sunny',
		};
		
		console.log(weather[today]);
    Notice how today isn't in quotes? This is how we specify that it is a variable instead of a key.

    What would happen if we tried the same thing but used dot notation?

    
		const today = 'wednesday';
		
		const weather = {
			monday: 'sunny',
			tuesday: 'rainy',
			wednesday: 'sunny',
		};
		
		console.log(weather.today);
	
		prints undefined

    So how do we know which one to use?

    According to Airbnb's styleguide
    I tend to agree with AirBnb, and find myself defaulting to dot notation unless I have a reason to use brackets.
    Basically, use dot notation unless you're doing something complicated!
